## 1️⃣ 修复段落首行缩进垂直偏移问题

### 问题描述
启用首行缩进时，虽然文字水平缩进了两个字符，但整行文字会稍微向上移动，导致与其他段落不对齐。

### 根本原因
`textIndent` CSS 属性在某些情况下会影响元素的布局，特别是当元素不是块级元素时。

### 修复方案
在 `useContextMenu.ts` 的 `applyParagraphSettings` 函数中添加了块级元素检查：

```typescript
// 【关键】确保元素是块级元素，防止 textIndent 影响垂直位置
if (element.style.display !== 'block' && element.style.display !== 'flex') {
  element.style.display = 'block';
}
```

### 修复效果
- ✅ 首行缩进只影响水平位置，不再影响垂直位置
- ✅ 所有段落的垂直对齐保持一致
- ✅ 首行缩进、悬挂缩进等功能正常工作

## 3️⃣ 重构表格边框样式功能

### 功能升级
从原来的统一边框设置升级为**10种精细化边框样式控制**，参考 UEditor 的实现。

### 新增边框样式类型

| 边框样式 | 说明 | 应用场景 |
|---------|------|---------|
| **无框线** | 移除所有边框 | 制作无边框表格 |
| **所有框线** | 表格和所有单元格都有边框 | 标准表格样式 |
| **外侧框线** | 仅表格外边框 | 简洁的表格样式 |
| **内部框线** | 仅单元格之间的边框 | 分隔单元格内容 |
| **内部横框线** | 仅水平方向的内部边框 | 行分隔样式 |
| **内部竖框线** | 仅垂直方向的内部边框 | 列分隔样式 |
| **上框线** | 仅表格顶部边框 | 标题下划线效果 |
| **下框线** | 仅表格底部边框 | 表格底部分隔 |
| **左框线** | 仅表格左侧边框 | 左侧强调 |
| **右框线** | 仅表格右侧边框 | 右侧强调 |

#### 3. 边框应用逻辑
**文件：`useContextMenu.ts`**

重写了 `applyTableSettings` 函数，实现了精确的边框控制：

```typescript
// 先清除所有边框
table.style.border = 'none';
cells.forEach(cell => {
  (cell as HTMLElement).style.border = 'none';
  // ... 清除各方向边框
});

// 根据边框样式类型应用边框
switch (settings.borderStyleType) {
  case 'inner':
    // 内部框线 - 精确控制每个单元格的边框
    cells.forEach((cell) => {
      const cellEl = cell as HTMLElement;
      const row = cellEl.parentElement as HTMLTableRowElement;
      const cellIndex = Array.from(row.cells).indexOf(cellEl as HTMLTableCellElement);
      const rowIndex = Array.from(table.querySelectorAll('tr')).indexOf(row);
      
      // 不是第一行，添加上边框
      if (rowIndex > 0) {
        cellEl.style.borderTop = borderValue;
      }
      // ... 其他方向的边框控制
    });
    break;
  // ... 其他样式类型
}
```

1. **选中表格**：右键点击表格

6. **应用设置**：点击"应用"按钮
