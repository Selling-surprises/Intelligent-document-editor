# 版本 3.11.0 功能总结

## 🎉 新增功能概览

本次更新（v3.11.0）在v3.10.0的基础上，新增了移动端适配功能，让导出的HTML文件能够更好地在移动设备上显示。

---

## 📱 移动端适配功能

### 核心特性

1. **可选的移动端适配**
   - 通过开关控制是否启用移动端适配
   - 默认开启，可根据需要关闭
   - 灵活满足不同使用场景

2. **独立的移动端背景图片**
   - 支持为移动端单独设置背景图片
   - 可以上传本地图片或使用在线图片URL
   - 移动端背景在屏幕宽度≤768px时自动显示

3. **移动端样式优化**
   - 自动调整字体大小（H1: 1.5em, H2: 1.25em, H3: 1.1em）
   - 优化内边距（body: 10px, container: 20px）
   - 调整容器圆角（4px）

---

## 🎴 链接卡片功能（v3.10.0）

### 核心特性

1. **双模式链接插入**
   - 文本链接：传统的超链接样式
   - 卡片链接：现代化的卡片展示

2. **灵活的卡片配置**
   - 必填：链接地址、卡片标题
   - 可选：卡片描述、卡片图片

3. **精美的视觉效果**
   - 圆角边框设计
   - 悬停动画效果
   - 响应式布局

## 🎯 使用场景

### 场景1：多设备访问的文档

**需求**：文档需要在桌面端和移动端都能良好显示

**解决方案**：
```
1. 启用移动端适配
2. 上传桌面端背景图片（横向）
3. 上传移动端背景图片（竖向）
4. 使用链接卡片展示重要链接
5. 导出HTML
```

**效果**：
- 桌面端显示横向背景，卡片横向布局
- 移动端显示竖向背景，卡片纵向布局
- 字体大小自动适配设备

### 场景2：仅桌面端使用的文档

**需求**：文档只在桌面端查看，不需要移动端优化

**解决方案**：
```
1. 关闭移动端适配
2. 上传桌面端背景图片
3. 使用文本链接或卡片链接
4. 导出HTML
```

**效果**：
- 导出的HTML不包含移动端样式
- 文件体积略小
- 桌面端显示完美

### 场景3：移动端优先的文档

**需求**：主要在移动端查看，桌面端为辅

**解决方案**：
```
1. 启用移动端适配
2. 重点设置移动端背景图片
3. 使用响应式卡片链接
4. 调整内容透明度
5. 导出HTML
```

**效果**：
- 移动端显示效果最佳
- 桌面端也能正常显示
- 响应式布局自动适配

## 💡 最佳实践

### 1. 背景图片选择

**桌面端背景**：
- 推荐尺寸：1920x1080 或更高
- 推荐格式：JPG、PNG、WebP
- 推荐内容：横向构图，色彩柔和

**移动端背景**：
- 推荐尺寸：1080x1920 或更高
- 推荐格式：JPG、PNG、WebP
- 推荐内容：竖向构图，简洁明了

### 2. 链接使用策略

**文本链接**：
- 适合内联引用
- 适合简单的URL分享
- 不需要特别强调的链接

**卡片链接**：
- 适合推荐资源
- 适合重要的外部链接
- 需要展示预览信息的链接

### 3. 移动端适配策略

**启用适配**：
- 文档需要在移动端查看
- 需要优化移动端显示效果
- 需要不同的移动端背景

**禁用适配**：
- 文档只在桌面端查看
- 不需要移动端优化
- 希望减小文件体积

## 📈 性能优化建议

### 1. 图片优化

- 使用WebP格式减小文件体积
- 压缩图片到合适的大小
- 桌面端背景：< 2MB
- 移动端背景：< 1MB

### 2. 内容优化

- 合理使用卡片链接（不要过多）
- 优化图片数量和大小
- 使用适当的内容透明度

### 3. 加载优化

- 使用CDN加速图片加载
- 使用HTTPS协议的图片URL
- 测试不同网络环境下的加载速度

## 🔧 技术实现

### 移动端适配实现

```typescript
// 类型定义
export interface EditorSettings {
  pageTitle: string;
  favicon: string;
  backgroundImage: string;
  opacity: number;
  enableMobileAdaptation: boolean;
  mobileBackgroundImage: string;
}

// 导出HTML时的条件渲染
@media (max-width: 768px) {
  ${settings.enableMobileAdaptation ? `
    body {
      ${settings.mobileBackgroundImage 
        ? `background: url(${settings.mobileBackgroundImage});` 
        : ''}
      padding: 10px;
    }
    
    .container {
      padding: 20px;
      border-radius: 4px;
    }
    
    h1 { font-size: 1.5em; }
    h2 { font-size: 1.25em; }
    h3 { font-size: 1.1em; }
  ` : ''}
}
```

### 链接卡片实现

```typescript
// 链接数据类型
export interface LinkData {
  url: string;
  text: string;
  type: 'text' | 'card';
  title?: string;
  description?: string;
  image?: string;
}

// 卡片HTML生成
if (linkData.type === 'card') {
  linkHtml = `
    <div class="link-card" contenteditable="false" 
         onclick="window.open('${linkData.url}', '_blank')">
      ${cardImage ? `<img src="${cardImage}" alt="${cardTitle}" />` : ''}
      <div>
        <div>${cardTitle}</div>
        ${cardDescription ? `<div>${cardDescription}</div>` : ''}
        <div>${linkData.url}</div>
      </div>
    </div>
  `;
}
```
