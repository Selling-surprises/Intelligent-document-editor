# 导出目录功能调试指南

## 🔍 问题诊断步骤

如果您导出的HTML文件没有显示目录，请按照以下步骤进行诊断：

---

## 步骤1：检查是否添加了标题

### 如何添加标题

1. 在编辑器中输入或选中一段文字
2. 点击工具栏中的**"标题"下拉菜单**
3. 选择"标题 1"、"标题 2"或"标题 3"

### 验证标题是否添加成功

- 标题文字应该变大
- 标题文字应该变成蓝色
- 左侧的悬浮目录应该显示该标题

**重要提示**：
- ❌ 只是把文字加粗或放大，**不是**标题
- ✅ 必须使用工具栏的"标题"下拉菜单来创建标题

---

## 步骤2：查看导出时的控制台信息

### 打开控制台

1. 在浏览器中按 **F12** 键
2. 切换到 **Console（控制台）** 标签
3. 点击 **导出** 按钮

### 查看调试信息

导出时，控制台会显示以下信息：

```
=== 开始导出 ===
原始content长度: 1234
原始content前100字符: <h1>第一章</h1><p>这是内容...</p>
tempDiv创建成功
找到的标题数量: 3
headingElements: NodeList(3) [h1, h2, h2]
标题 0: {text: "第一章", level: 1, tagName: "H1", id: "heading-0"}
标题 1: {text: "1.1 简介", level: 2, tagName: "H2", id: "heading-1"}
标题 2: {text: "1.2 背景", level: 2, tagName: "H2", id: "heading-2"}
exportHeadings数组长度: 3
是否生成目录: 是
exportHeadings数组: [{id: "heading-0", text: "第一章", level: 1}, ...]
=== 导出HTML的目录部分 ===
<body>
  
  <div class="floating-toc" id="floatingToc">
    <button class="toc-toggle" onclick="toggleToc()" title="切换目录">☰</button>
    <h2>目录</h2>
    <ul>
      <li class="level-1">
        <a href="#heading-0" onclick="scrollToHeading('heading-0')">第一章</a>
      </li>
      ...
    </ul>
  </div>
  
=== 目录部分结束 ===
```

### 诊断结果

#### 情况A：找到的标题数量 = 0

**问题**：文档中没有标题

**解决方法**：
1. 返回编辑器
2. 使用工具栏的"标题"下拉菜单添加标题
3. 确保标题文字变大且变成蓝色
4. 重新导出

#### 情况B：找到的标题数量 > 0，但导出的HTML没有目录

**问题**：可能是浏览器问题或HTML生成问题

**解决方法**：
1. 查看"=== 导出HTML的目录部分 ==="的输出
2. 如果这部分是空的，说明条件判断有问题
3. 截图控制台信息并反馈

#### 情况C：导出的HTML有目录代码，但看不到目录

**问题**：目录隐藏在左侧边缘外

**解决方法**：
1. 打开导出的HTML文件
2. 将鼠标移到页面**最左侧边缘**
3. 查找蓝色的"☰"按钮
4. 点击按钮显示目录

---

## 步骤3：检查导出的HTML文件

### 用文本编辑器打开HTML文件

1. 右键点击导出的HTML文件
2. 选择"打开方式" → "记事本"或其他文本编辑器
3. 搜索 `floating-toc`

### 检查结果

#### 如果找到了 `floating-toc`

说明目录代码已经生成，只是隐藏了。

**查找这段代码**：
```html
<div class="floating-toc" id="floatingToc">
  <button class="toc-toggle" onclick="toggleToc()" title="切换目录">☰</button>
  <h2>目录</h2>
  <ul>
    <li class="level-1">
      <a href="#heading-0" onclick="scrollToHeading('heading-0')">第一章</a>
    </li>
  </ul>
</div>
```

**解决方法**：
- 在浏览器中打开HTML文件
- 将鼠标移到页面左侧边缘
- 或者点击蓝色"☰"按钮

#### 如果没有找到 `floating-toc`

说明目录没有生成。

**可能的原因**：
1. 文档中没有标题
2. 导出时出现错误

**解决方法**：
1. 查看控制台的"找到的标题数量"
2. 如果是0，说明没有添加标题
3. 如果大于0，说明代码有bug，请反馈

---

## 步骤4：测试目录功能

### 创建测试文档

1. 清空编辑器
2. 输入以下内容：

```
第一章
这是第一章的内容

1.1 简介
这是简介的内容

1.2 背景
这是背景的内容

第二章
这是第二章的内容
```

3. 选中"第一章"，点击工具栏"标题"→"标题 1"
4. 选中"1.1 简介"，点击工具栏"标题"→"标题 2"
5. 选中"1.2 背景"，点击工具栏"标题"→"标题 2"
6. 选中"第二章"，点击工具栏"标题"→"标题 1"

### 验证标题

- 左侧悬浮目录应该显示4个标题
- 标题文字应该变大且变成蓝色

### 导出并测试

1. 按F12打开控制台
2. 点击导出按钮
3. 查看控制台输出，应该显示"找到的标题数量: 4"
4. 打开导出的HTML文件
5. 将鼠标移到左侧边缘，应该看到目录滑出

## 常见问题解答

### Q1：我添加了标题，但控制台显示"找到的标题数量: 0"

**A**：您可能没有正确添加标题。

**检查方法**：
1. 查看控制台的"原始content前100字符"
2. 看看是否包含 `<h1>`、`<h2>` 或 `<h3>` 标签
3. 如果没有，说明标题没有正确创建

**正确的标题HTML**：
```html
<h1>这是一级标题</h1>
<h2>这是二级标题</h2>
<h3>这是三级标题</h3>
```

**错误的标题HTML**：
```html
<p style="font-size: 24px; font-weight: bold;">这不是标题</p>
<div><b>这也不是标题</b></div>
```

### Q2：控制台显示有标题，但导出的HTML没有目录代码

**A**：这是一个bug，请提供以下信息反馈：

1. 控制台的完整输出（截图）
2. 浏览器名称和版本
3. 操作系统

### Q3：导出的HTML有目录代码，但看不到目录

**A**：目录默认隐藏在左侧边缘外。

**显示方法**：
- **方法1**：将鼠标移到页面最左侧边缘，目录会自动滑出
- **方法2**：查找并点击蓝色的"☰"按钮（在页面左侧中间位置）
- **方法3**：修改HTML文件，将 `translateX(-240px)` 改为 `translateX(0)`

### Q4：我看不到蓝色的"☰"按钮

**A**：按钮可能被遮挡或不明显。

**查找方法**：
1. 最大化浏览器窗口
2. 滚动到页面中间位置
3. 仔细查看页面最左侧边缘
4. 按钮有蓝色光晕闪烁动画

**如果还是看不到**：
1. 按F12打开开发者工具
2. 在Elements标签中搜索 `toc-toggle`
3. 右键点击该元素 → "Scroll into view"

## 技术支持

如果按照以上步骤仍然无法解决问题，请提供以下信息：

### 必需信息

1. **控制台完整输出**
   - 从"=== 开始导出 ==="到"=== 目录部分结束 ==="的所有内容
   - 截图或复制文本

2. **浏览器信息**
   - 浏览器名称（Chrome、Firefox、Safari、Edge等）
   - 浏览器版本号

3. **操作系统**
   - Windows、macOS、Linux等
   - 版本号

4. **问题描述**
   - 具体遇到了什么问题
   - 期望的结果是什么
   - 实际的结果是什么

### 可选信息

1. **导出的HTML文件**
   - 如果可以，提供导出的HTML文件
   - 或者提供HTML文件的关键部分（搜索 `floating-toc`）

2. **编辑器内容截图**
   - 显示您添加的标题
   - 显示左侧的悬浮目录
