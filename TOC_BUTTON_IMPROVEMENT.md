# 目录按钮交互优化说明 v3.11.1

## 📋 改进概述

优化了导出HTML文件中的目录按钮交互方式，使其更加直观和符合用户习惯。

---

## ✨ 改进内容

### 改进前（v3.11.0）
```
[☰] ← 目录按钮（固定显示）
 ↓ 点击
[☰] [目录面板] ← 两个按钮同时显示
     - 标题1
     - 标题2
```

**问题**：
- ❌ 目录展开后，目录按钮仍然显示
- ❌ 两个按钮同时存在，视觉混乱
- ❌ 用户可能不知道该点击哪个按钮来关闭目录

### 改进后（v3.11.1）
```
[☰] ← 目录按钮
 ↓ 点击
[目录面板 ✕] ← 目录按钮消失，折叠按钮出现
 - 标题1
 - 标题2
 - 标题3
 ↓ 点击 ✕
[☰] ← 目录按钮重新出现
```

**优势**：
- ✅ 点击目录按钮后，按钮会暂时消失
- ✅ 目录面板右上角显示折叠按钮（✕）
- ✅ 点击折叠按钮，目录隐藏，目录按钮重新出现
- ✅ 避免两个按钮同时显示，视觉更清晰
- ✅ 折叠按钮位置更符合用户操作习惯

## 🔧 技术实现

### 1. CSS样式

#### 目录展开按钮
```css
.toc-toggle {
  position: fixed;
  left: 0;
  top: 50%;
  transform: translateY(-50%);
  opacity: 1;
  transition: all 0.3s;
}

.toc-toggle.hidden {
  opacity: 0;
  pointer-events: none; /* 隐藏时不响应鼠标事件 */
}
```

#### 目录折叠按钮
```css
.toc-collapse {
  position: absolute;
  right: 10px;
  top: 10px;
  width: 32px;
  height: 32px;
  background: rgba(67, 97, 238, 0.95);
  border-radius: 6px;
}
```

#### 目录面板
```css
.floating-toc {
  padding-top: 60px; /* 为折叠按钮留出空间 */
}
```

### 2. HTML结构

```html
<!-- 目录展开按钮 -->
<button class="toc-toggle" id="tocToggle" onclick="toggleToc()">☰</button>

<!-- 目录面板 -->
<div class="floating-toc" id="floatingToc">
  <!-- 折叠按钮 -->
  <button class="toc-collapse" onclick="toggleToc()">✕</button>
  
  <h2>目录</h2>
  <ul>
    <li><a href="#heading1">标题1</a></li>
    <li><a href="#heading2">标题2</a></li>
  </ul>
</div>
```

### 3. JavaScript逻辑

```javascript
let tocVisible = false;

function toggleToc() {
  const toc = document.getElementById('floatingToc');
  const toggleBtn = document.getElementById('tocToggle');
  
  tocVisible = !tocVisible;
  
  if (tocVisible) {
    // 显示目录，隐藏展开按钮
    toc.classList.add('visible');
    toggleBtn.classList.add('hidden');
  } else {
    // 隐藏目录，显示展开按钮
    toc.classList.remove('visible');
    toggleBtn.classList.remove('hidden');
  }
}
```

## 📝 使用方法

### 在导出的HTML中使用

1. **打开导出的HTML文件**
   - 在浏览器中打开导出的HTML文件

2. **展开目录**
   - 在左侧中间位置看到蓝色的目录按钮（☰）
   - 点击按钮
   - 目录按钮消失，目录面板从左侧滑出
   - 目录面板右上角显示折叠按钮（✕）

3. **浏览目录**
   - 点击目录中的标题，页面会平滑滚动到对应位置
   - 目录面板保持打开状态

4. **折叠目录**
   - 点击目录面板右上角的折叠按钮（✕）
   - 目录面板隐藏
   - 目录按钮重新出现在左侧

## 🎯 设计理念

### 1. 避免视觉混乱
- 同一时间只显示一个按钮
- 展开按钮和折叠按钮不会同时出现
- 用户清楚地知道当前应该点击哪个按钮

### 2. 符合用户习惯
- 折叠按钮（✕）位于面板右上角，符合常见的关闭按钮位置
- 展开按钮（☰）位于左侧边缘，符合侧边栏的常见位置
- 按钮图标清晰易懂（☰ = 菜单，✕ = 关闭）

### 3. 流畅的动画
- 按钮使用opacity过渡，淡入淡出效果
- 目录面板使用transform过渡，滑动效果
- 所有动画时长统一为0.3秒，保持一致性

### 4. 无障碍设计
- 按钮有明确的title属性，提供提示信息
- 使用pointer-events: none确保隐藏的按钮不响应鼠标事件
- 按钮大小适中，易于点击（40px × 40px 和 32px × 32px）

## 🔍 技术细节

### 按钮状态管理

使用简单的布尔值`tocVisible`管理状态：
- `false`：目录隐藏，展开按钮显示
- `true`：目录显示，展开按钮隐藏，折叠按钮显示

### CSS类切换

使用`classList.add()`和`classList.remove()`方法：
- `.visible`：控制目录面板的显示/隐藏
- `.hidden`：控制展开按钮的显示/隐藏

### 动画实现

使用CSS transition实现平滑动画：
- `opacity`：按钮淡入淡出
- `transform`：目录面板滑动
- `transition: all 0.3s`：统一的动画时长
