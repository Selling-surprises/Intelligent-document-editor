# v3.12.9 卡片链接修复测试指南

## 📋 测试概述

本文档提供详细的测试步骤，用于验证v3.12.9版本中卡片链接的修复。

## 🧪 测试1：导出HTML中的编辑按钮

### 目标
验证导出的HTML文件中不显示编辑和删除按钮。

### 测试步骤

#### 步骤1：准备测试内容
1. 打开离线Word文档编辑器
2. 在编辑器中输入一些文本
3. 点击工具栏的"插入链接"按钮

#### 步骤2：插入多个卡片链接
1. 选择"卡片链接"选项
2. 输入第一个链接：`https://www.baidu.com`
3. 点击"自动获取"按钮
4. 等待获取完成，点击"插入"
5. 重复步骤2-4，插入至少3个不同的卡片链接

#### 步骤3：验证编辑器中的按钮
1. 将鼠标悬停在第一个卡片上
2. **预期结果**：应该显示"编辑"和"删除"按钮
3. 将鼠标移开
4. **预期结果**：按钮应该隐藏
5. 对其他卡片重复步骤1-4

#### 步骤4：导出HTML
1. 点击右侧设置面板的"导出HTML"按钮
2. **预期结果**：显示"导出成功"提示
3. 找到下载的HTML文件（文件名为设置的页面标题）

#### 步骤5：验证导出的HTML
1. 在浏览器中打开导出的HTML文件
2. 将鼠标悬停在第一个卡片上
3. **预期结果**：不应该显示任何按钮
4. 对其他卡片重复步骤2-3
5. 点击卡片内容区域
6. **预期结果**：应该在新标签页打开链接

#### 步骤6：检查HTML源代码（可选）
1. 在浏览器中右键点击页面，选择"查看页面源代码"
2. 搜索`card-actions`
3. **预期结果**：不应该找到任何`card-actions`元素
4. 搜索`editLinkCard`
5. **预期结果**：不应该找到`editLinkCard`函数
6. 搜索`deleteLinkCard`
7. **预期结果**：不应该找到`deleteLinkCard`函数

### 测试结果

| 检查项 | 预期结果 | 实际结果 | 状态 |
|--------|----------|----------|------|
| 编辑器中显示按钮 | ✅ 显示 | | |
| 编辑器中按钮可用 | ✅ 可用 | | |
| 导出HTML不显示按钮 | ✅ 不显示 | | |
| 导出HTML可点击链接 | ✅ 可点击 | | |
| 源代码无card-actions | ✅ 无 | | |
| 源代码无editLinkCard | ✅ 无 | | |
| 源代码无deleteLinkCard | ✅ 无 | | |

## 🧪 测试2：标题行插入卡片

### 目标
验证在标题行插入卡片时，卡片会自动转换为正文格式。

### 测试步骤

#### 步骤1：创建标题
1. 打开离线Word文档编辑器
2. 在编辑器中输入文本："这是一个H1标题"
3. 选中这段文本
4. 点击工具栏的"H1"按钮
5. **预期结果**：文本变为H1标题样式（大字体、加粗）

#### 步骤2：在标题行插入卡片
1. 将光标放在标题文本的末尾
2. 按Enter键，光标应该还在标题行
3. 点击工具栏的"插入链接"按钮
4. 选择"卡片链接"选项
5. 输入链接：`https://www.github.com`
6. 点击"自动获取"按钮
7. 等待获取完成，点击"插入"

#### 步骤3：验证卡片格式
1. 检查插入的卡片
2. **预期结果**：卡片应该使用正常的卡片样式，不是大字体
3. 检查卡片的边框、内边距、字体大小
4. **预期结果**：应该与在正文中插入的卡片样式一致

#### 步骤4：检查目录
1. 如果编辑器中有目录面板，检查目录内容
2. **预期结果**：目录中应该只有"这是一个H1标题"
3. **预期结果**：目录中不应该有卡片的内容

#### 步骤5：导出并验证
1. 点击"导出HTML"按钮
2. 在浏览器中打开导出的HTML文件
3. 检查卡片的样式
4. **预期结果**：卡片样式正常，不是标题样式
5. 检查目录（如果有）
6. **预期结果**：目录中不包含卡片内容

### 测试结果

| 检查项 | 预期结果 | 实际结果 | 状态 |
|--------|----------|----------|------|
| 卡片使用正常样式 | ✅ 正常 | | |
| 卡片不是大字体 | ✅ 不是 | | |
| 目录不包含卡片 | ✅ 不包含 | | |
| 导出HTML样式正常 | ✅ 正常 | | |
