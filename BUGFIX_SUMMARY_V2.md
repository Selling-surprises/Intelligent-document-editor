### 1. 表格选择和工具栏显示逻辑改进 🔧

**用户反馈**：
> 表格可以用鼠标选中一格、一行或者一列，只有处于选中状态才会有表格工具栏，删除表格可以只选择选中的表格

**问题分析**：
- 原实现：只要点击表格就显示工具栏
- 用户期望：只有选中单元格时才显示工具栏
- 缺少功能：无法清空单元格内容

**解决方案**：
1. 改进选择检测逻辑，检查是否真正选中了单元格
2. 添加"清空选中单元格"功能
3. 保留"删除整个表格"功能

**修改文件**：
- `src/components/editor/TableToolbar.tsx`

**新增功能**：
- ✨ 清空选中单元格内容
- ✨ 更精确的工具栏显示逻辑

### 2. 导出HTML文件没有悬浮目录 🔧

**用户反馈**：
> 导出的html文件没有左侧悬浮目录功能（已启用目录）

**问题分析**：
- 之前的修复只处理了标题ID的问题
- JavaScript代码在条件判断外，导致可能的错误
- 当没有标题时，JavaScript会尝试访问不存在的元素

**解决方案**：
1. 将JavaScript代码也放在条件判断内
2. 只在有标题时生成目录相关的JavaScript
3. 添加空值检查，防止错误

**修改文件**：
- `src/pages/Editor.tsx`

**改进点**：
- ✅ JavaScript代码只在有标题时生成
- ✅ 添加了空值检查
- ✅ 避免了潜在的运行时错误

## 📝 技术细节

### 表格工具栏改进

**之前的代码**：
```typescript
// 只检查是否在表格内
while (node && node !== editor) {
  if (node instanceof HTMLTableElement) {
    table = node;
    break;
  }
  node = node.parentNode;
}
```

**改进后的代码**：
```typescript
// 同时检查表格和单元格
let cell: HTMLTableCellElement | null = null;

while (node && node !== editor) {
  if (!cell && (node instanceof HTMLTableCellElement)) {
    cell = node;
  }
  if (node instanceof HTMLTableElement) {
    table = node;
    break;
  }
  node = node.parentNode;
}

// 只有当在表格单元格内时才显示工具栏
if (table && cell) {
  setSelectedTable(table);
  setVisible(true);
}
```

## 💡 经验总结

### 问题诊断
1. **用户反馈要仔细分析**：用户说"已启用目录"但导出没有目录，需要深入检查代码逻辑
2. **条件判断要完整**：不仅HTML要条件生成，相关的JavaScript也要条件生成
3. **样式问题要全面检查**：修复一个地方后，要检查其他类似的地方

### 代码质量
1. **空值检查很重要**：即使在条件判断内，也要添加空值检查
2. **选择器逻辑要精确**：不能只检查父元素，要检查具体的目标元素
3. **容器和内容要分离**：布局类应该在容器上，样式类在内容上

### 用户体验
1. **功能要符合直觉**：表格工具栏应该在选中时显示，而不是点击时显示
2. **样式要统一**：相同类型的组件应该有相同的样式
3. **错误要避免**：导出的HTML应该是完全可用的，不能有JavaScript错误
