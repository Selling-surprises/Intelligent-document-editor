# 目录功能说明

## 📚 目录是什么？

目录（Table of Contents，简称TOC）是文档编辑器自动生成的导航结构，用于快速定位和跳转到文档的不同章节。

---

## 🎯 目录包含什么？

### 包含的内容

目录**只包含**以下标题元素：

1. **一级标题（H1）** - 最大的标题
2. **二级标题（H2）** - 次级标题
3. **三级标题（H3）** - 三级标题
4. **四级标题（H4）** - 四级标题
5. **五级标题（H5）** - 五级标题
6. **六级标题（H6）** - 最小的标题

### 如何创建标题？

在工具栏中选择标题级别：

```
工具栏 → 标题下拉菜单 → 选择 H1/H2/H3/H4/H5/H6
```

**示例**：
- 选择"H1" → 创建一级标题
- 选择"H2" → 创建二级标题
- 选择"H3" → 创建三级标题

---

## ❌ 目录不包含什么？

### 不包含的内容

目录**不包含**以下内容：

1. ❌ **普通文本** - 正文内容
2. ❌ **链接** - 文本链接或卡片链接
3. ❌ **图片** - 插入的图片
4. ❌ **表格** - 插入的表格
5. ❌ **列表** - 有序或无序列表
6. ❌ **代码块** - 代码内容
7. ❌ **其他元素** - 除标题外的所有元素

---

## 🔗 关于链接卡片

### 为什么链接卡片不在目录中？

**原因**：
1. **链接卡片不是标题**
   - 链接卡片是用于展示外部链接的组件
   - 它不是文档结构的一部分
   - 它不代表文档的章节

2. **目录的作用**
   - 目录用于展示文档的**结构**
   - 目录用于快速**导航**到不同章节
   - 链接卡片不是章节，不需要导航

3. **设计原则**
   - 保持目录简洁清晰
   - 只显示真正的章节标题
   - 避免混淆内容和结构

## 📝 示例说明

### 示例文档

```
# 第一章：介绍          ← 会出现在目录中（H1标题）

这是一段普通文本。      ← 不会出现在目录中

## 1.1 背景            ← 会出现在目录中（H2标题）

这是背景介绍。         ← 不会出现在目录中

[链接卡片]             ← 不会出现在目录中（不是标题）
标题：参考资料
描述：相关链接
URL：https://example.com

### 1.1.1 历史         ← 会出现在目录中（H3标题）

这是历史介绍。         ← 不会出现在目录中
```

### 生成的目录

```
目录
├─ 第一章：介绍 (H1)
│  ├─ 1.1 背景 (H2)
│  │  └─ 1.1.1 历史 (H3)
```

**注意**：
- ✅ 标题出现在目录中
- ❌ 普通文本不出现
- ❌ 链接卡片不出现

## 🎨 目录的视觉效果

### 目录位置

- **桌面端**：左侧浮动面板
- **移动端**：顶部可折叠面板

### 目录功能

1. **展开/折叠**
   - 点击目录标题可以展开或折叠
   - 节省屏幕空间

2. **点击跳转**
   - 点击目录项可以跳转到对应标题
   - 平滑滚动效果

3. **层级显示**
   - 不同级别的标题有不同的缩进
   - 清晰显示文档结构

## 💡 使用建议

### 如何组织文档结构？

1. **使用标题划分章节**
   ```
   # 第一章
   ## 第一节
   ### 第一小节
   ```

2. **保持标题层级合理**
   - 不要跳级（如从H1直接到H3）
   - 保持层级清晰

3. **标题要简洁明了**
   - 使用简短的标题
   - 准确描述章节内容

4. **合理使用链接卡片**
   - 在相关章节中插入链接卡片
   - 作为补充资料，不作为章节标题

## 🔍 技术实现

### 目录生成逻辑

```typescript
// 只选择标题元素
const headingElements = editor.querySelectorAll('h1, h2, h3, h4, h5, h6');

// 为每个标题生成目录项
headingElements.forEach((element, index) => {
  const id = `heading-${index}`;
  element.id = id;
  
  newHeadings.push({
    id,
    level: parseInt(element.tagName[1]),  // 获取标题级别（1-6）
    text: element.textContent || '',      // 获取标题文本
    element: element as HTMLElement,
  });
});
```

**说明**：
- 使用`querySelectorAll('h1, h2, h3, h4, h5, h6')`选择所有标题
- 不包括其他元素（如div、a、span等）
- 链接卡片使用`<div class="link-card">`，不是标题元素

## ❓ 常见问题

### Q1：为什么我的链接卡片出现在目录中？

**A**：正常情况下不会出现。如果出现，可能是：
1. 链接卡片内部包含了标题标签（不应该发生）
2. 您手动在链接卡片中添加了标题标签

**解决方法**：
- 检查链接卡片的HTML结构
- 确保链接卡片只使用div、span等非标题元素

---

### Q2：如何让某个内容出现在目录中？

**A**：将该内容设置为标题：
1. 选中文本
2. 在工具栏选择标题级别（H1-H6）
3. 目录会自动更新

---

### Q3：如何隐藏某个标题，不让它出现在目录中？

**A**：目前不支持隐藏标题。如果不想某个内容出现在目录中：
1. 不要将它设置为标题
2. 使用加粗或其他格式代替标题样式

---

### Q4：目录可以手动编辑吗？

**A**：不可以。目录是自动生成的，会根据文档中的标题实时更新。

**如果需要修改目录**：
- 修改文档中的标题文本
- 目录会自动同步更新

---

### Q5：导出的HTML文件也有目录吗？

**A**：是的！导出的HTML文件包含：
1. ✅ 悬浮目录面板
2. ✅ 展开/折叠功能
3. ✅ 点击跳转功能
4. ✅ 平滑滚动效果

## 📊 目录 vs 链接卡片

### 功能对比

| 特性 | 目录 | 链接卡片 |
|------|------|----------|
| **作用** | 文档导航 | 外部链接展示 |
| **内容** | 文档标题 | 外部网页信息 |
| **位置** | 左侧面板 | 文档内容中 |
| **生成方式** | 自动生成 | 手动插入 |
| **可编辑** | 不可编辑 | 可编辑 |
| **点击效果** | 跳转到标题 | 打开外部链接 |

### 使用场景

**目录**：
- ✅ 快速浏览文档结构
- ✅ 跳转到特定章节
- ✅ 了解文档大纲

**链接卡片**：
- ✅ 引用外部资料
- ✅ 推荐相关链接
- ✅ 展示参考资源

```markdown
[链接卡片：教程视频]

❌ 不推荐：
[链接卡片：官方文档]  （没有章节标题）
[链接卡片：教程视频]
```

## 📚 总结

### 关键要点

1. **目录只包含标题**
   - H1、H2、H3、H4、H5、H6
   - 不包括其他任何元素

2. **链接卡片不是标题**
   - 链接卡片用于展示外部链接
   - 不会出现在目录中
   - 这是正常的设计

3. **目录自动生成**
   - 根据文档中的标题自动生成
   - 实时更新
   - 不可手动编辑

4. **合理组织文档**
   - 使用标题划分章节
   - 在章节中插入链接卡片
   - 保持结构清晰
