## ✨ 新增功能

### 1. 链接卡片自动获取网页信息

#### 功能描述
用户在插入卡片链接时，只需输入链接地址，点击"自动获取"按钮，系统会自动获取网页的标题、描述和图片，大大简化了卡片创建流程。

#### 核心特性
- 🤖 **一键获取**：输入URL后一键获取网页信息
- 🌐 **智能解析**：支持Open Graph和Twitter Card元数据
- 🖼️ **路径处理**：自动转换相对路径图片为绝对路径
- 💬 **友好提示**：提供清晰的加载状态和错误提示
- 🔄 **手动修改**：自动获取后仍可手动修改内容

#### 技术实现
- 使用CORS代理（allorigins.win）解决跨域问题
- 支持多种元数据格式（Open Graph、Twitter Card、标准HTML）
- 智能处理各种图片URL格式（绝对路径、相对路径、协议相对路径）

#### 使用方法
1. 点击"插入链接"按钮
2. 选择"卡片链接"类型
3. 输入链接地址
4. 点击"自动获取"按钮
5. 等待系统填充信息
6. 可选：手动修改自动获取的内容
7. 点击"插入链接"完成

## 🔧 问题修复

### 1. 插入链接后多出空行

#### 问题描述
在插入链接卡片后，编辑器中会多出一个空行，影响文档排版。

#### 根本原因
HTML字符串中包含多余的换行符和空格，导致`insertHTML`命令插入了额外的空白节点。

#### 解决方案
将卡片HTML从多行格式压缩为单行格式，移除所有不必要的换行符和空格。

#### 修改前
```javascript
linkHtml = `
  <div class="link-card" ...>
    <div class="card-actions" ...>
      ...
    </div>
  </div>
`;
```

#### 修改后
```javascript
linkHtml = `<div class="link-card" ...><div class="card-actions" ...>...</div></div>`;
```

---

### 2. 编辑和删除按钮显示问题

#### 问题描述
部分用户反馈插入的链接卡片没有显示编辑和删除按钮。

#### 根本原因
HTML中的换行符导致按钮元素被解析为独立的文本节点，影响了CSS样式的应用。

#### 解决方案
通过压缩HTML为单行，确保按钮元素正确嵌套在卡片容器中，CSS悬停样式能够正常工作。

## 🎯 最佳实践

### 1. 使用自动获取功能

**推荐做法**：
```
1. 输入链接地址
2. 点击"自动获取"
3. 检查自动填充的内容
4. 必要时手动修改
5. 插入链接
```

**不推荐做法**：
```
1. 完全依赖自动获取（某些网站可能失败）
2. 不检查自动获取的内容
3. 忽略错误提示
```

---

### 2. 处理获取失败

**失败原因**：
- 网站没有设置元数据
- 网站限制爬取
- 网络问题
- CORS代理限制

**解决方案**：
- 手动填写标题和描述
- 使用网站的favicon作为图片
- 简化描述内容
- 稍后重试

## 🔒 安全性说明

### CORS代理使用

**注意事项**：
- ⚠️ 仅用于获取公开的网页信息
- ⚠️ 不发送用户凭证或敏感信息
- ⚠️ 代理服务可能记录请求

**安全措施**：
- ✅ 使用HTTPS连接
- ✅ 仅获取元数据
- ✅ 不获取完整网页内容
- ✅ 不发送Cookie或Token
