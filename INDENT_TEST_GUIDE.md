# 缩进功能快速测试指南

## 🎯 测试目标

验证增加/减少缩进功能是否正常工作，包括工具栏按钮和键盘快捷键。

---

## 📋 快速测试清单（5分钟）

### 测试1：工具栏按钮 - 增加缩进（1分钟）

**步骤**：
1. 在编辑器中输入一段文本："这是一段测试文本"
2. 选中这段文本
3. 在工具栏中找到缩进按钮（位于列表按钮后面）
4. 点击"增加缩进"按钮（向右箭头图标）
5. 观察文本是否向右缩进

**预期结果**：
- ✅ 文本向右缩进约40px
- ✅ 按钮响应灵敏
- ✅ 缩进效果立即显示

---

### 测试2：工具栏按钮 - 减少缩进（1分钟）

**步骤**：
1. 继续使用上一步已缩进的文本
2. 保持文本选中状态
3. 点击"减少缩进"按钮（向左箭头图标）
4. 观察文本是否向左移动

**预期结果**：
- ✅ 文本向左移动，恢复原位
- ✅ 按钮响应灵敏
- ✅ 缩进效果立即显示

---

### 测试3：键盘快捷键 - Tab键（1分钟）

**步骤**：
1. 在编辑器中输入新的文本："测试Tab键缩进"
2. 选中这段文本
3. 按下`Tab`键
4. 观察文本是否向右缩进

**预期结果**：
- ✅ 文本向右缩进
- ✅ 快捷键响应灵敏
- ✅ 效果与工具栏按钮一致

---

### 测试4：键盘快捷键 - Shift+Tab（1分钟）

**步骤**：
1. 继续使用上一步已缩进的文本
2. 保持文本选中状态
3. 按下`Shift + Tab`键
4. 观察文本是否向左移动

**预期结果**：
- ✅ 文本向左移动，恢复原位
- ✅ 快捷键响应灵敏
- ✅ 效果与工具栏按钮一致

---

### 测试5：列表缩进（1分钟）

**步骤**：
1. 点击工具栏的"无序列表"按钮
2. 输入三个列表项：
   - 项目1
   - 项目2
   - 项目3
3. 将光标放在"项目2"上
4. 按`Tab`键
5. 观察列表层级变化

**预期结果**：
- ✅ "项目2"变为"项目1"的子列表
- ✅ 列表层级正确
- ✅ 缩进效果明显

## 🔍 详细测试步骤

### 测试A：多行同时缩进

**目的**：验证多行文本可以同时缩进

**步骤**：
1. 在编辑器中输入多行文本：
   ```
   第一行文本
   第二行文本
   第三行文本
   ```
2. 选中所有三行
3. 按`Tab`键
4. 观察所有行是否同时缩进
5. 按`Shift + Tab`键
6. 观察所有行是否同时减少缩进

**预期结果**：
- ✅ 所有选中的行同时缩进
- ✅ 所有选中的行同时减少缩进
- ✅ 缩进量一致

---

### 测试B：多层级缩进

**目的**：验证可以创建多层级缩进

**步骤**：
1. 输入一段文本
2. 按`Tab`键一次（第一层缩进）
3. 再按`Tab`键一次（第二层缩进）
4. 再按`Tab`键一次（第三层缩进）
5. 观察缩进层级

**预期结果**：
- ✅ 每次按Tab都增加一层缩进
- ✅ 缩进层级清晰可见
- ✅ 可以创建多层级结构

---

### 测试C：有序列表缩进

**目的**：验证有序列表的缩进功能

**步骤**：
1. 点击工具栏的"有序列表"按钮
2. 输入三个列表项：
   1. 第一项
   2. 第二项
   3. 第三项
3. 将光标放在"第二项"上
4. 按`Tab`键
5. 观察列表编号变化

**预期结果**：
- ✅ "第二项"变为"第一项"的子列表
- ✅ 列表编号自动调整
- ✅ 层级结构正确

---

**步骤**：

## 🐛 常见问题排查

### Q1：点击按钮没有反应？

**排查步骤**：
1. 检查是否选中了文本
2. 检查浏览器控制台是否有错误
3. 刷新页面重试

**解决方案**：
- 确保选中文本后再点击按钮
- 清除浏览器缓存
- 使用最新版本的浏览器

---

### Q2：Tab键不起作用？

**排查步骤**：
1. 检查光标是否在编辑器内
2. 检查是否有其他快捷键冲突
3. 尝试使用工具栏按钮

**解决方案**：
- 确保光标在编辑器内
- 关闭可能冲突的浏览器扩展
- 使用工具栏按钮作为替代

---

### Q3：缩进效果不明显？

**排查步骤**：
1. 检查缩进是否真的生效
2. 使用开发者工具查看HTML
3. 检查CSS样式

**解决方案**：
- 多次按Tab键增加缩进
- 检查浏览器缩放比例
- 确保没有自定义CSS覆盖

---

### Q4：列表缩进后编号错乱？

**排查步骤**：
1. 检查列表结构
2. 使用开发者工具查看HTML
3. 尝试重新创建列表

**解决方案**：
- 这是正常行为，子列表会重新编号
- 使用无序列表避免编号问题
- 手动调整列表结构

## 💡 测试技巧

### 技巧1：使用开发者工具

```
1. 按F12打开开发者工具
2. 选择Elements标签
3. 找到缩进的元素
4. 查看margin-left或padding-left属性
```

### 技巧2：测试边界情况

```
1. 测试空文本缩进
2. 测试单个字符缩进
3. 测试超长文本缩进
4. 测试特殊字符缩进
```

### 技巧3：对比测试

```
1. 创建两个相同的段落
2. 一个使用工具栏按钮缩进
3. 一个使用Tab键缩进
4. 对比两者效果是否一致
```

### 技巧4：连续操作测试

```
1. 连续按Tab键多次
2. 观察缩进是否累加
3. 连续按Shift+Tab多次
4. 观察缩进是否减少到0
```
