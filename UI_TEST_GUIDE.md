# UI 美化与移动端优化 - 快速测试指南

## 测试环境

### 桌面端测试
- **推荐分辨率**: 1920×1080, 1366×768, 1440×900
- **浏览器**: Chrome, Firefox, Safari, Edge

### 移动端测试
- **推荐设备**: iPhone 12/13/14, Samsung Galaxy S21/S22
- **模拟器**: Chrome DevTools 设备模拟
- **推荐分辨率**: 375×667, 414×896, 430×932

## 一、桌面端测试清单

### 1.1 视觉效果测试

#### ✅ 色彩系统
1. 打开编辑器，观察整体配色
   - 标题应显示蓝紫渐变效果
   - 背景应为柔和的浅色渐变
   - 按钮应有现代化的配色

2. 测试悬浮效果
   - 鼠标悬浮在编辑区域上，应有轻微抬升效果
   - 鼠标悬浮在表格上，阴影应增强
   - 鼠标悬浮在链接卡片上，应有顶部渐变条出现

#### ✅ 阴影系统
1. 观察编辑器主体
   - 应有柔和的大阴影（shadow-xl）
   - 悬浮时阴影应增强

2. 观察表格
   - 表格应有中等阴影（shadow-md）
   - 悬浮时阴影应变为 shadow-lg

#### ✅ 圆角设计
1. 检查各组件圆角
   - 编辑器主体：大圆角（rounded-2xl）
   - 按钮：中等圆角（rounded-lg）
   - 输入框：小圆角（rounded-md）

### 1.2 功能测试

#### ✅ 工具栏
1. 顶部工具栏应完整显示
2. 所有按钮应可点击
3. 下拉菜单应正常工作

#### ✅ 状态栏
1. 底部状态栏应显示
2. 字符数和字数应实时更新
3. 当前字体和字号应正确显示
4. 图标应正确显示

#### ✅ 侧边栏
1. 左侧目录应显示（如有标题）
2. 点击目录项应平滑滚动到对应位置
3. 目录项应有悬浮效果

## 二、移动端测试清单

### 2.1 布局测试

#### ✅ 响应式布局
1. 打开 Chrome DevTools（F12）
2. 切换到设备模拟模式（Ctrl+Shift+M）
3. 选择 iPhone 12 Pro（390×844）
4. 观察布局变化：
   - 顶部标题应变小
   - 桌面端工具栏应隐藏
   - 底部应出现移动端工具栏
   - 编辑区域内边距应减小

#### ✅ 底部工具栏
1. 底部工具栏应固定在屏幕底部
2. 应显示快捷操作按钮：
   - 加粗（B）
   - 斜体（I）
   - 下划线（U）
   - 无序列表（•）
   - 有序列表（1）
3. 应显示撤销/重做按钮
4. 应显示"更多"按钮（⋯）

#### ✅ 更多工具抽屉
1. 点击"更多"按钮
2. 应从底部弹出抽屉
3. 抽屉应包含：
   - 标题选项（H1, H2, H3）
   - 对齐选项（左、中、右）
   - 插入选项（链接、图片、表格、代码）
   - 颜色选项
4. 点击任意选项后抽屉应关闭

### 2.2 触摸优化测试

#### ✅ 点击区域
1. 使用触摸模式（Chrome DevTools 中启用）
2. 测试所有按钮：
   - 最小点击区域应为 44×44px
   - 按钮应易于点击
   - 不应出现误触

#### ✅ 滚动体验
1. 在编辑区域滚动
   - 应流畅无卡顿
   - 顶部栏应保持固定
   - 底部工具栏应保持固定

### 2.3 文字编辑测试

#### ✅ 输入体验
1. 点击编辑区域
2. 输入文字
   - 字号应适中（15px）
   - 行高应舒适（1.8）
   - 字体应清晰

#### ✅ 格式化
1. 选中文字
2. 点击底部工具栏的格式化按钮
   - 加粗应生效
   - 斜体应生效
   - 下划线应生效

## 三、跨设备测试

### 3.1 平板测试（768px - 1024px）

#### iPad (768×1024)
1. 切换到 iPad 模拟
2. 检查布局：
   - 应显示桌面端工具栏
   - 应显示桌面端状态栏
   - 不应显示移动端工具栏
   - 内边距应为中等（p-12）

### 3.2 大屏手机测试（414px - 430px）

#### iPhone 14 Pro Max (430×932)
1. 切换到 iPhone 14 Pro Max
2. 检查布局：
   - 应显示移动端工具栏
   - 顶部标题应为小号
   - 按钮应为小尺寸
   - 内边距应为小号（p-8）

### 3.3 小屏手机测试（375px）

#### iPhone SE (375×667)
1. 切换到 iPhone SE
2. 检查布局：
   - 所有内容应正常显示
   - 不应出现横向滚动
   - 按钮应可点击
   - 文字应清晰可读

## 五、性能测试

### 5.1 加载性能

#### ✅ 首次加载
1. 清除浏览器缓存
2. 刷新页面
3. 观察加载时间
   - 应在 2 秒内完成

4. 测试操作流畅度
   - 输入应无延迟
   - 滚动应流畅
   - 格式化应即时生效

## 十、测试技巧

### 10.1 Chrome DevTools 快捷键
- `F12`: 打开开发者工具
- `Ctrl+Shift+M`: 切换设备模拟模式
- `Ctrl+Shift+C`: 元素选择器
- `Ctrl+Shift+P`: 命令面板

### 10.2 设备模拟
1. 打开 DevTools
2. 点击设备工具栏图标
3. 选择预设设备或自定义尺寸
4. 启用触摸模拟
5. 调整网络速度（可选）

### 10.3 性能分析
1. 打开 DevTools
2. 切换到 Performance 标签
3. 点击录制按钮
4. 执行操作
5. 停止录制
6. 分析性能报告
