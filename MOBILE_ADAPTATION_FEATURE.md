# 移动端适配功能说明

## 功能概述

离线Word文档编辑器现在支持导出HTML时的移动端适配选项，可以为移动端设备提供优化的显示效果。

---

## 功能特点

### 1. 可选的移动端适配
- 通过开关控制是否启用移动端适配
- 默认开启，可根据需要关闭
- 关闭后导出的HTML将只保留桌面端样式

### 2. 独立的移动端背景图片
- 支持为移动端单独设置背景图片
- 可以上传本地图片或使用在线图片URL
- 移动端背景图片在屏幕宽度小于768px时自动显示

### 3. 移动端优化样式
- 自动调整内边距和外边距
- 优化标题字体大小
- 调整容器圆角和布局

---

## 使用方法

### 启用移动端适配

1. **打开设置面板**
   - 点击右上角的"设置"按钮

2. **找到移动端适配选项**
   - 在设置面板中向下滚动
   - 找到"移动端适配"开关

3. **启用适配**
   - 打开"移动端适配"开关
   - 开关变为蓝色表示已启用

### 上传移动端背景图片

#### 方法1：本地上传

```
1. 确保"移动端适配"开关已打开
2. 在"移动端背景图片"区域选择"本地上传"标签
3. 点击文件选择按钮
4. 选择一张图片文件
5. 图片会自动上传并显示预览
```

#### 方法2：在线图片

```
1. 确保"移动端适配"开关已打开
2. 在"移动端背景图片"区域选择"在线图片"标签
3. 输入图片URL（例如：https://example.com/mobile-bg.jpg）
4. 点击"确定"按钮
5. 图片会自动加载并显示预览
```

### 移除移动端背景图片

```
1. 在移动端背景图片预览区域
2. 点击右上角的"移除"按钮
3. 背景图片将被清除
```

## 样式说明

### 桌面端样式（屏幕宽度 > 768px）

```css
body {
  background: 桌面端背景图片;
  padding: 20px;
}

.container {
  max-width: 900px;
  padding: 40px;
  border-radius: 8px;
}

h1 { font-size: 2em; }
h2 { font-size: 1.5em; }
h3 { font-size: 1.25em; }
```

### 移动端样式（屏幕宽度 ≤ 768px，启用适配时）

```css
body {
  background: 移动端背景图片（如果设置）;
  padding: 10px;
}

.container {
  padding: 20px;
  border-radius: 4px;
}

h1 { font-size: 1.5em; }
h2 { font-size: 1.25em; }
h3 { font-size: 1.1em; }
```

## 使用场景

### 场景1：不同设备使用不同背景

**需求**：桌面端使用横向背景图，移动端使用竖向背景图

**操作**：
```
1. 上传桌面端背景图片（横向，如1920x1080）
2. 启用移动端适配
3. 上传移动端背景图片（竖向，如1080x1920）
4. 导出HTML
```

**效果**：
- 在桌面浏览器中显示横向背景
- 在手机浏览器中显示竖向背景
- 背景图片自动适配设备

### 场景2：仅桌面端显示

**需求**：只在桌面端显示，移动端不需要特殊优化

**操作**：
```
1. 关闭"移动端适配"开关
2. 导出HTML
```

**效果**：
- 导出的HTML不包含移动端适配样式
- 在移动设备上显示桌面端样式
- 文件体积略小

### 场景3：移动端优先

**需求**：主要在移动端查看，桌面端为辅

**操作**：
```
1. 启用移动端适配
2. 上传适合移动端的背景图片
3. 桌面端背景可以留空或使用相同图片
4. 导出HTML
```

**效果**：
- 移动端显示效果最佳
- 桌面端也能正常显示
- 响应式布局自动适配

## 背景图片建议

### 桌面端背景图片

**推荐尺寸**：
- 1920x1080（Full HD）
- 2560x1440（2K）
- 3840x2160（4K）

**推荐格式**：
- JPG（适合照片）
- PNG（适合图形）
- WebP（体积小，质量高）

**推荐内容**：
- 横向构图
- 色彩柔和
- 不要太复杂

### 移动端背景图片

**推荐尺寸**：
- 1080x1920（竖屏）
- 1440x2560（高分辨率竖屏）
- 750x1334（iPhone标准）

**推荐格式**：
- JPG（适合照片）
- PNG（适合图形）
- WebP（体积小，质量高）

**推荐内容**：
- 竖向构图
- 色彩柔和
- 简洁明了

## 技术实现

### 媒体查询

```css
@media (max-width: 768px) {
  /* 移动端样式 */
}
```

### 条件渲染

```typescript
${settings.enableMobileAdaptation ? `
  /* 移动端适配样式 */
  body {
    ${settings.mobileBackgroundImage 
      ? `background: url(${settings.mobileBackgroundImage});` 
      : ''}
    padding: 10px;
  }
` : ''}
```

### 背景图片优先级

1. 移动端（屏幕 ≤ 768px）：
   - 如果设置了移动端背景图片，使用移动端背景
   - 如果没有设置，使用桌面端背景

2. 桌面端（屏幕 > 768px）：
   - 始终使用桌面端背景图片

## 常见问题

### Q1：移动端背景图片不显示？

**可能原因**：
- 图片URL不正确
- 图片不支持跨域访问
- 网络连接问题
- 屏幕宽度大于768px

**解决方法**：
1. 检查图片URL是否正确
2. 使用HTTPS协议的图片
3. 在移动设备或浏览器开发者工具的移动模式下测试
4. 尝试使用本地上传的图片

### Q2：关闭移动端适配后，移动端显示不正常？

**说明**：
- 这是正常现象
- 关闭适配后，移动端会显示桌面端样式
- 如果需要移动端优化，请启用移动端适配

### Q3：可以只设置移动端背景，不设置桌面端背景吗？

**回答**：
- 可以
- 桌面端会显示默认的渐变背景
- 移动端会显示设置的移动端背景

### Q4：移动端背景图片会影响桌面端吗？

**回答**：
- 不会
- 移动端背景只在屏幕宽度小于768px时显示
- 桌面端始终使用桌面端背景

### Q5：如何测试移动端效果？

**方法1：使用真实设备**
```
1. 导出HTML文件
2. 将文件传输到手机
3. 在手机浏览器中打开
```

**方法2：使用浏览器开发者工具**
```
1. 在浏览器中打开导出的HTML
2. 按F12打开开发者工具
3. 点击"切换设备工具栏"图标（或按Ctrl+Shift+M）
4. 选择移动设备型号
5. 查看移动端效果
```

## 最佳实践

### 1. 背景图片选择

✅ **推荐做法**：
- 使用高质量图片
- 桌面端和移动端使用不同构图
- 图片色彩柔和，不影响文字阅读
- 使用WebP格式减小文件体积

❌ **避免做法**：
- 使用过大的图片（超过5MB）
- 使用过于复杂的图片
- 使用低分辨率图片
- 使用过于鲜艳的颜色

### 2. 适配策略

✅ **推荐做法**：
- 默认启用移动端适配
- 为移动端设置专门的背景图片
- 测试不同设备的显示效果
- 调整内容透明度以确保可读性

❌ **避免做法**：
- 忽略移动端用户
- 使用相同的背景图片
- 不测试移动端效果
- 背景图片影响文字阅读

### 3. 性能优化

✅ **推荐做法**：
- 压缩图片文件
- 使用CDN加速图片加载
- 使用WebP格式
- 合理设置图片尺寸

❌ **避免做法**：
- 使用原始大小的图片
- 使用过多的背景图片
- 忽略加载速度
- 使用过大的文件
