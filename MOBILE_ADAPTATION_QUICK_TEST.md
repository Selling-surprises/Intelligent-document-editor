# 移动端适配功能快速测试指南

## 🚀 3分钟快速测试

### 测试1：启用移动端适配（1分钟）

```
1. 点击右上角的"设置"按钮
2. 向下滚动找到"移动端适配"选项
3. 确认开关是打开状态（蓝色）
4. 点击"导出为HTML文件"按钮
5. 在浏览器中打开导出的HTML文件
6. 按F12打开开发者工具
7. 按Ctrl+Shift+M切换到移动设备模式
8. 选择"iPhone 12 Pro"或其他移动设备
9. 观察页面显示效果
```

**预期结果**：
- ✅ 页面自动适配移动设备
- ✅ 字体大小适中
- ✅ 内边距合理
- ✅ 内容可读性好

---

### 测试2：上传移动端背景图片（1分钟）

```
1. 在设置面板中确保"移动端适配"开关已打开
2. 在"移动端背景图片"区域选择"本地上传"
3. 点击文件选择按钮
4. 选择一张竖向图片（建议1080x1920）
5. 等待图片上传完成
6. 导出HTML文件
7. 在移动设备模式下查看效果
```

**预期结果**：
- ✅ 移动端显示上传的背景图片
- ✅ 背景图片铺满整个屏幕
- ✅ 内容区域透明度正常
- ✅ 文字清晰可读

---

### 测试3：禁用移动端适配（1分钟）

```
1. 在设置面板中关闭"移动端适配"开关
2. 注意"移动端背景图片"选项消失
3. 导出HTML文件
4. 在移动设备模式下查看效果
```

**预期结果**：
- ✅ 移动端显示桌面端样式
- ✅ 字体大小与桌面端相同
- ✅ 布局与桌面端一致
- ✅ 不显示移动端背景图片

## 🐛 常见问题排查

### 问题1：移动端背景不显示
**检查**：
- [ ] 是否启用了移动端适配
- [ ] 是否上传了移动端背景图片
- [ ] 图片URL是否正确
- [ ] 是否在移动设备模式下查看（屏幕宽度≤768px）

### 问题2：移动端字体太大或太小
**说明**：
- 这是正常的适配效果
- 移动端字体会自动调整
- 如果觉得不合适，可以关闭移动端适配

### 问题3：桌面端显示了移动端背景
**检查**：
- [ ] 浏览器窗口宽度是否小于768px
- [ ] 尝试最大化浏览器窗口
- [ ] 检查是否在移动设备模式下

### 问题4：移动端适配开关无法关闭
**解决**：
- 刷新页面重试
- 检查浏览器控制台是否有错误
- 尝试重新打开编辑器

### 问题5：导出的HTML在真实手机上显示不正常
**检查**：
- [ ] 手机浏览器是否支持现代CSS
- [ ] 图片URL是否可以在手机上访问
- [ ] 网络连接是否正常
- [ ] 尝试使用不同的浏览器

## 💡 测试技巧

1. **使用浏览器开发者工具**
   - 快捷键：F12 或 Ctrl+Shift+I
   - 切换设备模式：Ctrl+Shift+M
   - 可以自定义设备尺寸

2. **测试不同网络环境**
   - 在开发者工具中可以模拟慢速网络
   - Network标签 → Throttling → Slow 3G

3. **测试不同屏幕方向**
   - 在设备模式下可以旋转屏幕
   - 点击旋转图标切换横屏/竖屏

4. **使用真实设备测试**
   - 模拟器无法完全替代真实设备
   - 建议至少在一台真实设备上测试

5. **保存测试结果**
   - 截图保存测试结果
   - 记录发现的问题
   - 便于后续改进
