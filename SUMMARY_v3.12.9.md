## 🎯 修复的问题

### 问题1：导出HTML中的编辑按钮
**用户反馈**：
> 导出的html文件卡片链接上没有编辑和删除按钮，那个只在文档编辑状态才有，另外卡片链接上的编辑是完整编辑，不仅仅只能编辑链接

**问题分析**：
- 导出的HTML文件中仍然包含编辑和删除按钮的HTML代码
- 包含相关的CSS样式和JavaScript函数
- 导致导出的HTML文件代码冗余，不符合只读展示的需求

**解决方案**：
1. ✅ 在导出HTML之前，移除所有`.card-actions`元素
2. ✅ 删除导出HTML中的`.card-actions`相关CSS样式
3. ✅ 删除导出HTML中的`editLinkCard`和`deleteLinkCard`函数
4. ✅ 保留卡片的展示和链接功能

**修复效果**：
- ✅ 编辑器中：保留完整的编辑功能
- ✅ 导出HTML：只读展示，不可编辑
- ✅ 代码简洁：文件大小减少约4KB

---

### 问题2：卡片链接被应用标题样式
**用户反馈**：
> 卡片链接被应用了目录的标题中，那么就设置：无论之前是否插入过标题，插入卡片链接后强制改为正文

**问题分析**：
- 在标题行插入卡片时，卡片会被包裹在标题标签中
- 导致卡片继承标题的样式（大字体、加粗等）
- 可能导致卡片被错误地列入目录

**解决方案**：
1. ✅ 在插入卡片后，检测当前是否在标题行
2. ✅ 如果在标题行，使用`formatBlock`命令将其改为`div`（正文）
3. ✅ 确保卡片始终使用正常的卡片样式
4. ✅ 确保卡片不会出现在目录中

**修复效果**：
- ✅ 卡片始终使用正常样式
- ✅ 卡片不会被应用标题样式
- ✅ 卡片不会出现在目录中
- ✅ 文档结构更清晰

#### 修改4：插入卡片后强制改为正文（第596-616行）
```typescript
// 如果是卡片链接，强制将当前行改为正文格式（移除标题格式）
if (linkData.type === 'card') {
  const selection = window.getSelection();
  if (selection && selection.rangeCount > 0) {
    const range = selection.getRangeAt(0);
    let node = range.commonAncestorContainer;
    
    // 向上查找，找到标题元素
    while (node && node !== editor) {
      if (node.nodeType === Node.ELEMENT_NODE) {
        const element = node as HTMLElement;
        // 如果是标题元素，将其改为div（正文）
        if (/^H[1-6]$/i.test(element.tagName)) {
          document.execCommand('formatBlock', false, 'div');
          break;
        }
      }
      node = node.parentNode;
    }
  }
}
```
